indexR in Drill

支持数据类型

  • int
  • bigint, alias: long
  • float
  • double
  • varchar, alias: string
  • date. Format: 2017-02-17.
  • datetime, alias: timestamp. Format: 2017-02-17 12:30:03 or 2017-02-17T12:30:03. Supports millisecond, e.g. 2017-02-17 12:30:03.214.

indexR建表

Drill 查询引擎无法直接建表，需要通过index-tool

创建schema

以下为schema 官网说明:

  • schema - [必须项] the schema of table.
  • columns - 表的列名
  • location - 表数据存储位置，建议使用配置文件中位置。这个不做配置
  • mode - 默认使用vlt，也可以使用basic，basic提供10：1的压缩比。vlt有更快的查询扫描速度。
  • agg - 预聚合。
  1. grouping - 默认为false,设置为true时，必须配置dims和metrics。
    聚合会将相同纬度的数据合并在一起。   它可以极大地减少OLAP场景中的数据大小，因为我们不关心独立事件，而是关注维度组的整个图像。
  • dims - 聚合维度
  • metrics - 聚合指标
  1. sum
  2. min
  3. max
  4. first
  5. last
